OSCAL Security Assessment Results (SAR) Format

1.0.0-milestone2

The short name (file identifier) for this schema shall be oscal-sar. It is used internally when an identifier is called for, and may appear in file names of schema artifacts.

http://csrc.nist.gov/ns/oscal/1.0

The OSCAL Control SSP format can be used to describe the information typically specified in a system security plan, such as those defined in NIST SP 800-18.

The root of the OSCAL System Security Plan (SSP) format is system-security-plan.

Security Assessment Report (SAR): system-security-plan assembly

A system security plan, such as those described in NIST SP 800-18

The system-security-plan assembly has the following contents (in order):

  • Flag (required)
  • An assembly (zero or one)
  • An assembly (zero or one)
  • An assembly (zero or one)

Import System Security Plan: import-ssp assembly

Used to import the OSCAL profile representing the system's control baseline.

The import-ssp assembly has the following contents :

  • Flag (required)
  • An field (zero or one)

Scope of Assessment: scope assembly

.

The scope assembly has the following contents :

  • Flag (required)
  • An field (zero or one)
